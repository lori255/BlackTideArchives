# 1. 项目概述
## 1.1 一句话定位
玩家经营一家隐藏于雾港的异常收容机构，派遣调查员回收、研究并利用旧日异常物；每一次变强都伴随理智、污染和失控风险，机构会在收益与灾变之间持续博弈。
## 1.2 核心卖点
- 每件异常物都是一条可利用的危险规则：它能提供能力、资源或情报，也会制造收容事故和污染代价。
- 调查员不是普通角色卡，而是会升阶、异化、失控、被封存或转化为异常实体的长期资产。
- 任务不是单纯刷关，而是由异常报告、随机节点、风险检定、撤退选择和回收结果组成的轻 Roguelite 事件链。
- 克苏鲁负责宏观威胁与不可知代价，SCP 式档案负责微观异常规则，诡秘式升阶负责 RPG 成长结构。
- 网页端承载复杂 UI、档案、卡牌、图鉴、配置热更新和云存档，降低打开门槛。
## 1.3 目标体验
| 体验目标 | 说明 |
|---|---|
| 随开随玩 | 玩家打开网页即可收取、派遣、处理事故、查看战报，不需要安装客户端。 |
| 浅层可推进 | 几次点击就能完成资源收取、任务派遣、研究排队、异常物维护。 |
| 深层可研究 | 调查员路线、异常物组合、收容布局、污染控制、晋阶仪式均可形成策略深度。 |
| 强氛围但不重恐怖 | 以档案感、未知感、低压恐怖和黑色幽默为主，避免过度惊吓。 |
| 内容可持续扩展 | 异常物、案件、调查员特质、路线能力、收容事故可长期配置化扩充。 |

# 2. 世界观与题材融合
## 2.1 世界观基调
故事发生在 20 世纪初风格的架空沿海城市“雾港”。海雾、灯塔、旧教堂、地下档案室、失踪船员、异常潮汐和无法解释的录音共同构成世界基调。玩家所属的黑潮档案局表面是民间灾害记录机构，实际负责回收、收容和研究旧日异常。
## 2.2 三类设定的融合方式
| 来源灵感 | 在游戏中的作用 | 修改原则 |
|---|---|---|
| 克苏鲁 | 提供旧日存在、理智崩坏、禁忌知识、宏观污染、不可逆代价。 | 不堆砌神名，重点表现“知识有代价”和“世界逐渐失控”。 |
| SCP/异常收容 | 提供编号、危险等级、收容规则、事故报告、异常物图鉴、实验记录。 | 不使用现成 SCP 条目和名称，所有组织、编号、异常物规则原创。 |
| 诡秘式升阶 | 提供路线、材料、仪式、行为稳定、失控风险的 RPG 成长结构。 | 不照搬途径/序列/魔药名，改为“异常路线 + 晋阶协议 + 稳定行为”。 |

## 2.3 设计边界
- 不直接使用 SCP Foundation、具体 SCP 编号、SCP 站点条目和相关标识。
- 不直接复刻《诡秘之主》的路径、序列、魔药、人物、组织和专有名词。
- 可以借鉴结构思想：规则化异常、收容代价、路线升阶、仪式条件、失控风险。
- 所有可商业化内容应尽量原创命名、原创规则、原创图像、原创剧情。
# 3. 核心玩法与循环
## 3.1 总循环
```text
接收异常报告 → 选择回收任务 → 编队调查员 → 进入随机事件链 → 回收异常物/撤退/失败 → 收容或封存异常物 → 研究获得能力与材料 → 调查员晋阶 → 风险与污染上升 → 机构扩建与风险管理 → 解锁更高等级异常。
```
## 3.2 玩法深度分层
| 层级 | 玩家行为 | 系统反馈 |
|---|---|---|
| 轻操作层 | 收资源、派任务、领取报告、处理警报、查看异常状态。 | 即时资源、任务进度、状态变化、少量风险提示。 |
| 决策层 | 派谁去、带什么异常物、是否继续深入、是否撤退、是否牺牲资源稳定收容。 | 成功率、理智变化、污染变化、奖励稀有度、事故概率。 |
| 策略层 | 路线搭配、异常物组合、收容布局、研究路径、旧日苏醒控制。 | 长期强度、风险结构、稀有事件、版本/赛季目标。 |

## 3.3 核心乐趣判断
- 玩家获得强力异常物时，应同时产生“想用”和“不敢乱用”的心理。
- 调查员晋阶不应只是数值提升，而应改变任务选择、异常适配和失控风险。
- 收容事故不能只是惩罚，应转化为新的任务、档案、异常实体或稀有机会。
- 每次任务都应该让玩家至少做一次有代价的选择，而非纯自动结算。
- 玩家应能通过配置和理解规则降低风险，而不是完全被随机支配。
# 4. 第一版范围与非目标
## 4.1 MVP 必须包含
- 账号登录与云存档。
- 局本部主界面：资源、行动力、黑潮指数、旧日苏醒、日常任务。
- 任务系统：至少 12 个异常任务模板，支持随机节点、风险、奖励、撤退。
- 调查员系统：至少 8 名基础调查员，6 类属性，理智/污染/伤病/特质。
- 异常物系统：至少 20 件异常物，包含收容效果、装备效果、事故事件、研究产物。
- 异常路线系统：首版 4 条路线，每条至少 4 个等级与 8 个技能/被动。
- 收容区系统：房间、槽位、稳定性、事故、维护。
- 研究系统：异常物研究、路线材料、机构科技。
- 档案系统：异常物档案、任务报告、调查员履历。
- AI 美术资产流程：角色、异常物、场景、图标的生成、后处理、入库规范。
## 4.2 第一版不做
- 实时 PVP、强同步多人副本、玩家自由交易市场。
- 高复杂动作战斗、平台跳跃、实时弹幕、复杂走位。
- 大世界开放探索、3D 场景、复杂地形 TileMap。
- 完整移动端原生 App。
- 强剧情演出、大量配音、长篇主线文本。
- 完全自动生成并直接上线的 AI 美术，所有资源必须后处理和审核。
# 5. 核心系统需求
## 5.1 异常任务系统
异常任务是主要消耗行动力并产出异常物、材料、线索和污染的玩法。任务以节点链形式呈现，玩家在关键节点作出选择。
## 5.2 调查员系统
调查员是可培养、可升阶、可异化、可失控的核心角色资产。
## 5.3 异常物系统
异常物是游戏最核心的收集与规则系统。每件异常物都必须具备正面用途和负面代价。
## 5.4 异常路线与晋阶系统
异常路线是 RPG 成长核心，借鉴“路线、仪式、代价、失控”的结构，但全部改造为游戏系统。第一版建议先做 4 条路线：梦境、深海、血肉、密仪。
## 5.5 稳定行为与失控
调查员晋阶后需要通过符合路线特质的行为维持稳定度。稳定度过低会触发失控。失控不是单一死亡惩罚，而是新的风险与内容来源。
## 5.6 收容区系统
收容区是机构经营核心。它不是普通仓库，而是异常物风险管理场。
- 房间类型：普通收容室、低温收容室、梦境隔离室、深海压力舱、仪式封印室、观察室、研究室、医疗室、记忆清洗室。
- 关键指标：稳定性、污染度、维护成本、相邻冲突、事故概率、封存状态。
- 异常物之间可产生联动：强化、抵消、冲突、污染扩散、连锁事故。
- 玩家要在收益最大化和事故可控之间做布局与维护。

# 6. 页面与功能需求
| 页面 | 核心功能 | 第一版要求 |
|---|---|---|
| 登录/注册 | 邮箱/账号登录、游客转正、找回密码。 | 支持云存档绑定，登录后进入局本部。 |
| 局本部首页 | 资源、行动力、旧日苏醒、黑潮指数、任务入口、公告。 | 作为主 Dashboard，展示关键风险和待处理事项。 |
| 异常任务页 | 任务列表、任务详情、节点路线、编队、派遣。 | 参考概念图，必须是第一版核心页面。 |
| 任务执行页 | 节点事件、选择、检定、奖励、撤退。 | 支持半自动推进与关键选择。 |
| 调查员页 | 角色列表、属性、状态、装备、路线、晋阶。 | 支持筛选、编队、治疗、查看履历。 |
| 异常物页 | 图鉴、收容状态、装备、研究、封存。 | 每件异常物必须有档案页和规则描述。 |
| 收容区页 | 房间、槽位、稳定性、事故、维护。 | 支持房间升级、异常物移动、事故处理。 |
| 研究页 | 异常物研究、机构科技、路线配方。 | 支持队列与材料消耗。 |
| 晋阶仪式页 | 路线等级、材料、行为条件、风险预估。 | 晋阶前必须展示收益和失控概率。 |
| 档案页 | 任务报告、异常物档案、调查员履历、世界线索。 | 强化世界观与收集感。 |
| 设置页 | 音量、画质、账号、通知、隐私。 | Web 端常规设置。 |
